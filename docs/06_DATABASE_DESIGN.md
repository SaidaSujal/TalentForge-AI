# Database Design

## Database
- Production: PostgreSQL hosted on Neon.
- Development / Testing: Local PostgreSQL.
- **SQLite is strictly prohibited** for deployment to maintain JSONB, pgvector, and transactional DDL compatibility.

## ORM
- Use **SQLAlchemy 2.x** declarative mapping (`Mapped` and `mapped_column`).
- Use **Alembic** for schema migrations. All schema changes must be migration-based.
- Keep migration scripts clean, transactional, and sequential (e.g., `0001_enable_pgvector`, `0002_create_foundation_tables`, `0003_create_users_table`, `0004_create_hr_foundation_tables`).

## HR Foundation Schema (Phase 2.2)

### 1. Employees Table (`employees`)
Represents internal corporate personnel. Supports self-referential management hierarchy, tenant-scoped unique email constraints, and sensitive salary deferred loading.

| Column | Type | Nullable | Description / Constraint |
| :--- | :--- | :--- | :--- |
| `id` | UUID | No | Primary Key |
| `company_id` | UUID | No | Foreign Key to `companies.id` (CASCADE) |
| `user_id` | UUID | Yes | Foreign Key to `users.id` (SET NULL), Unique |
| `first_name` | VARCHAR(100) | No | First name |
| `last_name` | VARCHAR(100) | No | Last name |
| `email` | VARCHAR(255) | No | Corporate email address |
| `phone` | VARCHAR(50) | Yes | Contact phone number |
| `department` | VARCHAR(100) | No | Department name |
| `role` | VARCHAR(100) | No | Job title / role description |
| `salary` | NUMERIC(12, 2) | Yes | Sensitive salary (deferred/lazy-loaded) |
| `manager_id` | UUID | Yes | Self-referential FK to `employees.id` (SET NULL) |
| `status` | ENUM | No | `employee_status_enum` (active, suspended, terminated, onboarding) |
| `role_enum` | VARCHAR(50) | Yes | Optional enum representation |
| `hire_date` | DATE | Yes | Date of hire |
| `work_mode` | ENUM | Yes | `work_mode_enum` (remote, hybrid, office) |
| `experience_level` | ENUM | Yes | `experience_level_enum` (junior, mid, senior, lead, director) |
| `current_skills` | JSONB | Yes | Array of parsed skills, default `[]` |
| `target_role` | VARCHAR(100) | Yes | Target career progression role |
| `is_deleted` | BOOLEAN | No | Soft-delete flag (default `False`) |
| `created_by` | UUID | Yes | Audit field (creating user ID) |
| `updated_by` | UUID | Yes | Audit field (last updating user ID) |
| `created_at` | TIMESTAMPTZ | No | Server default `now()` |
| `updated_at` | TIMESTAMPTZ | No | Server default `now()` |

#### Constraints & Indexes
- **Unique Constraint**: `uq_employees_email_company` on `(email, company_id)`
- **Composite Indexes**:
  - `ix_employees_company_status` on `(company_id, status)`
  - `ix_employees_company_department` on `(company_id, department)`
  - `ix_employees_company_role` on `(company_id, role)`
  - `ix_employees_company_created_at` on `(company_id, created_at)`
- **Single-Column Indexes**: `ix_employees_user_id` (Unique), `ix_employees_manager_id`, `ix_employees_email`

### 2. Candidates Table (`candidates`)
Represents job applicants before hiring. Stores extracted profiles and AI matching scores.

| Column | Type | Nullable | Description / Constraint |
| :--- | :--- | :--- | :--- |
| `id` | UUID | No | Primary Key |
| `company_id` | UUID | No | Foreign Key to `companies.id` (CASCADE) |
| `first_name` | VARCHAR(100) | Yes | Extracted first name |
| `last_name` | VARCHAR(100) | Yes | Extracted last name |
| `email` | VARCHAR(255) | Yes | Extracted email address |
| `phone` | VARCHAR(50) | Yes | Extracted phone number |
| `status` | ENUM | No | `candidate_status_enum` (applied, screened, shortlisted, interviewing, offered, hired, rejected) |
| `experience_years`| NUMERIC(4, 1) | Yes | Parsed experience in years |
| `education` | TEXT | Yes | Academic profile summary |
| `current_role` | VARCHAR(150) | Yes | Current job title |
| `skills` | JSONB | Yes | Array of parsed skills, default `[]` |
| `match_score` | NUMERIC(5, 2) | Yes | AI match score (0.00 - 100.00) |
| `scorecard_json` | JSONB | Yes | AI assessment metrics / criteria scores |
| `suggested_questions` | JSONB | Yes | AI-generated candidate interview questions |
| `interview_invitation_text` | TEXT | Yes | Draft email generated by AI |
| `is_deleted` | BOOLEAN | No | Soft-delete flag (default `False`) |
| `created_by` | UUID | Yes | Audit field (creating user ID) |
| `updated_by` | UUID | Yes | Audit field (last updating user ID) |
| `created_at` | TIMESTAMPTZ | No | Server default `now()` |
| `updated_at` | TIMESTAMPTZ | No | Server default `now()` |

#### Constraints & Indexes
- **Check Constraint**: `chk_candidates_match_score` enforces `match_score >= 0.0 AND match_score <= 100.0`
- **Composite Indexes**:
  - `ix_candidates_company_status` on `(company_id, status)`
  - `ix_candidates_company_email` on `(company_id, email)`
  - `ix_candidates_company_created_at` on `(company_id, created_at)`
- **Single-Column Indexes**: `ix_candidates_company_id`, `ix_candidates_email`, `ix_candidates_status`

### 3. Resumes Table (`resumes`)
Stores metadata and parsed text content from uploaded resume documents.

| Column | Type | Nullable | Description / Constraint |
| :--- | :--- | :--- | :--- |
| `id` | UUID | No | Primary Key |
| `company_id` | UUID | No | Foreign Key to `companies.id` (CASCADE) |
| `candidate_id` | UUID | No | Foreign Key to `candidates.id` (CASCADE), Unique |
| `file_name` | VARCHAR(255) | No | Original uploaded filename |
| `file_path` | VARCHAR(500) | Yes | Temporary file storage path |
| `file_size` | INTEGER | No | Size in bytes |
| `mime_type` | VARCHAR(100) | No | Expected PDF or DOCX MIME types |
| `raw_text` | TEXT | No | Extracted text content from resume |
| `resume_hash` | VARCHAR(64) | No | SHA-256 hash of extracted text |
| `is_deleted` | BOOLEAN | No | Soft-delete flag (default `False`) |
| `created_by` | UUID | Yes | Audit field (creating user ID) |
| `updated_by` | UUID | Yes | Audit field (last updating user ID) |
| `created_at` | TIMESTAMPTZ | No | Server default `now()` |
| `updated_at` | TIMESTAMPTZ | No | Server default `now()` |

#### Constraints & Indexes
- **Unique Constraint**: `uq_resumes_candidate` on `candidate_id` (enforces 1-to-1 candidate to resume)
- **Composite Indexes**:
  - `ix_resumes_company_candidate` on `(company_id, candidate_id)` (Unique)
  - `ix_resumes_company_hash` on `(company_id, resume_hash)`
- **Single-Column Indexes**: `ix_resumes_company_id`, `ix_resumes_candidate_id`, `ix_resumes_resume_hash`

| Column | Type | Nullable | Description / Constraint |

## Vector Storage
- Use pgvector extension for chatbot knowledge bases.
- Table `policy_chunks` stores document split chunks, vector embeddings, and links back to `policy_documents`.

## Security Rules
- **Passwords**: Raw passwords must never be stored. Store password hashes using bcrypt.
- **Salary Loading**: Confidential salaries use SQLAlchemy `deferred` lazy loading to ensure they are omitted from general queries unless explicitly requested.
- **Tenant Isolation**: Query isolates data using composite indexes prefixed with `company_id`.
